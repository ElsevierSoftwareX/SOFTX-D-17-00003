The "Find" functionallity can be used to search for certain patterns in a sequence.

The search is always performed from top to bottom, starting from the current position of the cursor in the sequence. The option "Wrap Search" can be used to begin searching from the start of the sequence if the end was reached before a matching string was found. As soon as a match was found the sequence will be selected and marked in the editor.

By default the search input does only allow single nucliotide bases. For a more complex search, regular expressions can be used by enabeling the "Regular Expression" checkbox. Any combination of the constructs listed in the following table can be used to search for a given pattern.

Summary of regular-expression constructs

x[abc][^abc][a-zA-Z][a-d[m-p]][a-z&&[def]][a-z&&[^bc]][a-z&&[^m-p]].X?X*X+X{n}X{n,}X{n,m}XYX|Y(X)

d, e, or f (intersection)

To match e.g. all codons (tuples of length 3) ending in U, the regular expression ??U may be used. For tessera (tuples of length 4) starting with U and ending either in A or C the regular expression U??(A|C) applies.

For more information on regular expression constructs, please see the Java documentation on Patterns.
